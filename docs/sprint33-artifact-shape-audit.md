# Sprint 33 Artifact Shape Audit

Date: 2026-06-12

## Decision

PermitGraph should stay local-first and read one generated dashboard snapshot. The snapshot exporter can use existing repo-local aggregate artifacts now, but Sprint 33 needs to preserve or copy per-repo run artifacts into `.agent-permit` before the dashboard can support full evidence drilldown and proof pack export.

## Current Snapshot Shape

The frontend currently reads:

```text
generatedAt
source
runMeta
summary
savedViews
findings
artifactPreviews
traceSteps
policyControls
```

The current `findings` array is not raw scanner findings. It is one row per repository in the live validation set. That is useful for first-screen review, but it is not enough for full evidence drilldown.

## Trusted Field Groups

| Need | Existing fields |
| --- | --- |
| Repo/run identity | `repo_id`, `run_id`, `source`, `repo_path`, `validation_run_id`, `completed_at` |
| Permit outcome | `actual_permit_status`, `expected_permit_status`, `passed`, `live_validation_passed` |
| Risk counts | `findings_count`, `graph_paths_count`, `controls_count`, `actual_rule_ids` |
| Quality checks | `citation_check_passed`, `expectation_check_passed`, `status_check_passed`, `expected_rule_check_passed` |
| Cost/cache | `input_tokens`, `total_tokens`, `cached_tokens`, `cache_hit_ratio`, `model_calls`, `duration_seconds` |
| Repo freshness | `commit`, `commit_date`, `commit_message` from demo prep |
| Local scan metrics | `finding_severity_counts`, `rule_counts`, `available_artifacts`, `phoenix`, `langsmith`, `model` |

## Gaps

1. Per-repo artifact paths in `live-repo-validation-results.json` point to `/private/tmp/agent-permit-open-source-validation-20260607/...`.
2. Those temp directories are no longer present locally.
3. Repo-local live validation aggregate has counts and rule IDs, but not full raw findings, graph paths, permits, or Deep Agent reports for each repo.
4. `raw-findings.json` uses object shape `{ scan_run_id, findings }`, not a top-level array.
5. `graph-paths.json` uses object shape `{ scan_run_id, taxonomy, paths }`, not a top-level array.
6. The dashboard contract has no `repos`, `runs`, or `selectedRunId` collections yet, so run/repo switching would be awkward.
7. `artifactPreviews` only includes aggregate live validation JSON and report today.

## Sprint 33 Contract Implications

The next snapshot contract should add:

```text
repos[]
runs[]
selectedRunId
rows[]              # one review row per repo or finding, depending view mode
runDetails{}        # keyed by run id
artifactPreviews{}  # aggregate plus per-run preserved artifacts
decisionLog[]       # scanner -> graph -> permit -> Deep Agent -> final action
proofPack{}         # export manifest and included files
```

The exporter should first support the existing aggregate-only shape, then enrich rows when per-repo artifacts are available. Missing per-repo artifacts should be explicit in the snapshot, not silently hidden.

## Sprint 33 Update

`tools/export_dashboard_snapshot.py` now emits a versioned snapshot contract with repo/run collections and explicit artifact availability.

New live validation runs preserve per-repo artifacts under:

```text
.agent-permit/live-repo-validations/<validation_run_id>/repos/<repo_id>/<repo_run_id>/
```

The existing Sprint 23 aggregate remains partial because its referenced `/private/tmp` artifacts were already gone before Sprint 33.
